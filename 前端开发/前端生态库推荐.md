### 构建工具

- Vite（当前主流）
- Turbopack（Next.js力推）
- [Rsbuild](https://github.com/web-infra-dev/rsbuild)：一个由 Rspack 驱动的高性能构建工具。它提供了精心设计的默认配置，让你无需任何配置即可开始项目，同时完全利用了 Rspack 的卓越性能。
- [tsdown](https://github.com/rolldown/tsdown) ⭐：由 Rolldown 驱动的库捆绑器，为 TypeScript 和 JavaScript 库提供无缝且高效的打包方式。
  - 主要特点：
    - 简化的配置：`tsdown` 提供了针对库开发的合理默认配置，减少了复杂配置的需求。
    - 面向库的特性：与作为通用打包器的 Rolldown 不同，`tsdown` 专为构建库而优化。

### 开发辅助插件

- [vite-plugin-qrcode](https://github.com/svitejs/vite-plugin-qrcode)：服务器启动时显示二维码，方便移动设备访问本地开发环境。
- [plugin-web-update-notification](https://github.com/GreatAuk/plugin-web-update-notification)：检测网页更新并通知用户刷新页面，支持 Vite、Umijs 和 Webpack，使用版本号（如 git commit hash）进行比较。
- [vite-plugin-validate-env](https://github.com/Julien-R44/vite-plugin-validate-env)：Vite 插件用于验证环境变量，在构建或开发时验证环境变量，确保配置正确，支持内置验证器和标准模式（如 Zod）。
- [vite-plugin-mock-dev-server](https://github.com/pengzhanbo/vite-plugin-mock-dev-server)：为 Vite 提供 API 模拟开发服务器，支持多种文件类型和热更新。
- <br />

### 框架迁移

- [vue-ai-migrator](https://github.com/TylerDurden75/vue-ai-migrator)：最全面的 Vue 2 → Vue 3 迁移工具，支持基于 AST 的转换和 AI 集成。自动转换为 Composition API、脚本设置、路由器 4 和 Pinia。

## 基础通用能力

### JavaScript 工具库

- [es-toolkit](https://github.com/toss/es-toolkit)：一个现代 JavaScript 工具库，速度快 2-3 倍，体积小了 97%，是 lodash 的重大升级。
- [radashi](https://github.com/radashi-org/radashi)：Radash官方归档后的社区续命版，性能比Lodash快3-7倍，体积减少75%。
  - 采用ES2020全新算法，零`any`，错误优先风格，类型体验拉满。
  - 维护活跃， nightly 发版，RFC公开投票，适配最新前端生态。
- [pinyin-match](https://github.com/xmflswood/pinyin-match)：拼音匹配，具备分词、缩写、多音字匹配能力，支持繁体版

### 文档 / SEO

- [unhead](https://github.com/unjs/unhead) ⭐：Unhead 是一款文档`<head>`和模板管理器，致力于提升反应式 SSR JavaScript 框架开发者的体验、SEO 和性能。

### 数据校验

- [validator.js](https://github.com/validatorjs/validator.js)：通用字符串校验库，覆盖邮箱、手机号、URL、IP、长度、正则规则等高频验证场景。
  - 适用场景：表单验证、登录注册、URL 检查、手机号和邮箱格式校验。
  - 注意：适合做基础规则校验；复杂业务表单建议结合 `zod`、`yup` 或 `vee-validate`。

### 数字精度处理

- [decimal.js](https://github.com/MikeMcl/decimal.js) ⭐：高精度数字计算库，解决 JavaScript 原生浮点数精度问题，支持任意精度运算与大数字处理。
  - 注意：金额计算强烈推荐统一封装金额工具层，避免业务代码直接混用原生 Number。

### 加密与摘要

- [crypto-js](https://github.com/brix/crypto-js)：常用前端加密与摘要算法库，支持 MD5、SHA256、AES、HMAC 等。
  - 注意：前端加密主要用于传输和展示层安全增强，真正安全仍依赖 HTTPS、服务端签名校验和密钥管理。

## 应用开发能力

### 国际化

- [vue-i18n: Vue I18n for Vue 3](https://github.com/intlify/vue-i18n) ⭐
- [unplugin-vue-i18n](https://github.com/intlify/bundle-tools/tree/main/packages/unplugin-vue-i18n)

### 状态管理增强

- [pinia-colada](https://github.com/posva/pinia-colada) ⭐：专注服务端状态缓存，类似 TanStack Query 的 Pinia 风格方案。
  - **适用场景**：列表缓存、请求去重、自动刷新、乐观更新。
- [pinia-plugin-persistedstate](https://github.com/prazdevs/pinia-plugin-persistedstate)：最常用的 Pinia 持久化插件，可将 store 自动同步到 localStorage / sessionStorage / cookie。
  - 适用场景：登录态、用户配置、主题模式、多页签缓存。
- [pinia-plugin-persist](https://github.com/Seb-L/pinia-plugin-persist)：轻量级持久化方案，支持自定义存储策略。
  - 适用场景：对持久化逻辑有二次封装需求的项目。
- [pinia-plugin-debounce](https://github.com/posva/pinia-plugin-debounce)：为 actions 增加防抖能力，避免高频触发请求。
  - 适用场景：搜索输入、表单自动保存、实时筛选。
- [pinia-shared-state](https://github.com/wobsoriano/pinia-shared-state)：多标签页共享 Pinia 状态，基于 BroadcastChannel / localStorage 通信。
  - 适用场景：后台管理系统多标签同步、H5 多窗口同步。
- [pinia-undo](https://github.com/wobsoriano/pinia-undo)：为 Pinia Store 添加撤销 / 重做能力，跟踪状态变化并提供历史记录。
  - 适用场景：表单编辑器、低代码页面搭建、流程配置、画布类应用、AI 对话草稿回退。
- [pinia-orm](https://github.com/CodeDredd/pinia-orm)：为 Pinia 提供 ORM 能力，支持 Model、Relation、Query。
  - 适用场景：复杂业务数据管理，如 IM、审批流、组织架构、商品系统。

### 交互能力

- [vue3-touch-events](https://github.com/robinrodricks/vue3-touch-events)：Vue3 专用触控事件库，可在任意 HTML 元素上启用 tap、swipe、touch、hold、drag、zoom 等事件，一套语法同时支持 PC 鼠标与移动端触屏

### 布局与大屏适配

- [autofit.js](https://github.com/Auto-Plugin/autofit.js)：无框架依赖的通用缩放工具，通过计算容器与设计稿的比例，对整个页面进行缩放处理，核心逻辑是 `transform: scale(ratio)` 的全局应用。
- [vfit.js](https://github.com/v-plugin/vfit) ⭐：专为 Vue 3 设计的轻量方案，不仅提供全局缩放，更通过组件化思想解决**精细定位**问题，是"缩放+定位"的一体化方案。

### 列表与表格性能优化

- [vue-virtual-scroller](https://github.com/Akryum/vue-virtual-scroller)：虚拟滚动，顺畅渲染大型 Vue 列表，同时不放弃对用户界面的控制。
  - 滚动时不创建新节点，只更新内容
- [stk-table-vue](https://github.com/ja-plus/stk-table-vue)：一个基于 Vue 的高性能虚拟列表组件。

### 二维码 / 条形码 / 签名 / 打印

- [bwip-js](https://github.com/metafloor/bwip-js)：把专业级 PostScript 条形码生成器翻译成纯 JS 的库， 不依赖图片服务、不调用外部接口、不联网，在浏览器、Node.js、React、React Native、Electron 里直接生成标准条形码。
- [vue-qrcode-reader](https://github.com/gruhn/vue-qrcode-reader)：一套用于检测和解码二维码的 Vue.js 组件。
- [Vue-Print-Designer](https://github.com/0ldFive/Vue-Print-Designer)：一款可视化打印设计器，面向业务表单、标签、票据、快递单等场景，支持模板化、变量化，并提供静默打印与云打印能力，同时兼容多种导出/打印方式。
- [vue3-signature-pad](https://github.com/selemondev/vue3-signature-pad)：基于 HTML5 Canvas 实现的 Vue3 签名板组件

## 跨端与桌面

### 桌面端开发

- [Electron](https://github.com/electron/electron)：用 JavaScript、HTML 和 CSS 构建跨平台桌面应用
  - 当前最流行的桌面端方案
  - 特点：内置 Chromium + Node.js、可以直接用 React / Vue / Vite、生态成熟，插件多
  - 缺点：体积大、内存占用高
- [Tauri](https://github.com/tauri-apps/tauri)：构建更小、更快、更安全的桌面和移动应用，并配备网页前端。
  - Electron 的“轻量替代”
  - 特点：前端：HTML/CSS/JS（React/Vue）；后端：Rust；使用系统 WebView
  - 优点：体积小、性能更好、更安全（权限控制严格）
  - 适合：追求性能 + 体积的小工具、客户端
- [Flutter](https://github.com/flutter/flutter)：Google 的跨平台 UI 框架
  - 特点：用 Dart 写、自带渲染引擎（不是 Web），性能接近原生、UI 一致性强
  - 适合：对性能和 UI 要求高的应用

### 原生可执行方向

- [PerryTS/perry](https://github.com/PerryTS/perry)：一个用 Rust 编写的本地 TypeScript 编译器。
  - 直接把你的 TypeScript 编译成原生可执行文件——没有 Node.js，没有 Electron，也没有浏览器引擎。只是快速、小的二进制，能跑到任何地方。
